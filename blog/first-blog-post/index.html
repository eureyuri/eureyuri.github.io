First blog post! (and how I made it) v1

Hello, World!

This is my first blog post made with flask, peewee, and postgresql!
Shout out to this blog post I referenced to make this: https://charlesleifer.com/blog/how-to-make-a-flask-blog-in-one-hour-or-less/
Since the blog post was written to work with sqlite, I had to modify a little to work with postgresql. The reason for this is because sqlite runs in memory and stores data in a file. Since Heroku has an ephemeral file system, the data stored will be periodically cleared (!) and therefore, it would be necessary for us to implement the databse with postgres.
I thought of writing briefly about the switch from sqlite to postgresql since some of you may want to deploy to Heroku and have an actual working blog that doesn't erase your data.

Move to PostgreSQL

  1. Add the following import statements from playhouse.postgres_ext import * from dotenv import load_dotenv
  2. postgres_ext is for the searching capability, which I decided not to include so I will not really touch on it but you can read the documentation (http://docs.peewee-orm.com/en/latest/peewee/playhouse.html#postgres-ext) and make the necessary changes.
  3. The dotenv package is for loading in the DATABASEURL (as well as all the other credentials including ADMINPASSWORD and the flask SECRET_KEY).
  4. Next include the line DATABASE = os.environ['DATABASE_URL']
  5. This will get the DATABASE_URL from Heroku
  6. That's it for the Python part!
  7. We'll get our PostgreSQL addon on Heroku. On your command line run $ heroku addons:create heroku-postgresql:<PLAN_NAME>
  8. For the plan name I chose hobby-dev
  9. Create the database on Heroku. On your command line run $ heroku run python from your_app import db python your_app.py
  10. Now we are ready to deploy to Heroku $ git add . $ git commit -m "PostgreSQL " $ git push heroku master and that should be it!

  11. Notes: Accessing database on Heroku from command line $ heroku pg:psql When I ran this, I got the message:

        https://devcenter.heroku.com/articles/heroku-postgresql#local-setup```

This happens when the PosgreSQL versions do not match. You can check the version by

$ heroku pg:info and $ psql --version To install you can run the following command: $ brew install postgresql@12

Custom domain

  • Buy a domain. (I bought it at Name.com for free using Github's student package) $ heroku domains:add www.eureyuri.codes
  • If you have not verified your account, you will have to verify on Heroku by putting in your card info
  • On Name.com, click on "Manage DNS" and add a record with type CNAME, host *, and answer with what heroku gave you when you added your custom domain. It should be in the form somename.herokudns.com
  • Then host your website on your custom domain $ host www.eureyuri.codes
